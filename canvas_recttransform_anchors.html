The Root of All UI
The Canvas is the root container for all Unity UI. Every UI element must be a child of a Canvas. It defines how UI elements are rendered to the screen. Click a render mode to preview it in the interactive 3D scene.
Default Screen Space — Overlay
UI is drawn on top of everything, after the scene renders. It always covers the full screen and is not affected by any camera. Use this for HUDs, menus, and standard game UI.
Camera Screen Space — Camera
The Canvas is rendered by a specific camera at a set distance in front of it. UI elements are affected by camera perspective, rotation, and post-processing effects. Use this when UI should be overlaid but integrated with scene depth.
3D World Space
The Canvas behaves like a regular 3D object in the scene. It has physical positions, rotation, and scale, allowing it to appear behind or in front of world objects. Use this for character nameplates, in-world terminals, or floating health bars.

Pivot, Rotation, and Scaling
The pivot is the local origin of rotation and scale. By shifting the pivot, you control the anchor point around which the element rotates or grows. Click a preset to relocate the pivot, then rotate or scale.
